# 프로젝트 개요: 아키텍처최적화 서비스 소개 영상 제작

본 문서는 Gemini Veo를 활용하여 '아키텍처최적화 서비스'를 효과적으로 소개하기 위한 영상 제작 계획과 가이드라인을 담고 있습니다.

## 1. 프로젝트 목표
- **서비스 인지도 제고**: 아키텍처최적화 서비스의 필요성과 가치를 고객에게 명확히 전달
- **고품질 영상 제작**: Gemini Veo의 최신 생성 AI 기능을 활용하여 2~3분 분량의 전문적인 소개 영상 제작
- **일관성 확보**: 영상 전반에 걸쳐 일관된 비주얼과 톤앤매너 유지

## 2. 제작 단계 (Process)

### Phase 1: Planning & Scripting (기획 및 시나리오)
- **아이디어 브레인스토밍**: 서비스의 핵심 셀링 포인트(USP) 도출
- **스크립트 작성**: 내레이션 및 자막 텍스트 확정
- **스토리보드 구성**: 각 씬(Scene)별 비주얼 구성 및 애니메이션 흐름 설계

### Phase 2: Pilot Test (테스트 영상 제작)
- **목적**: Gemini Veo의 결과물 품질 확인 및 일관성 테스트
- **규모**: 1~2분 분량의 간단한 시나리오 기반 영상
- **주요 체크포인트**: 
    - 캐릭터 생성이 일관되게 유지되는가?
    - 배경 및 색감이 기획 의도와 맞는가?
    - 프롬프트 제어를 통해 원하는 연출이 가능한가?

### Phase 3: Production (메인 영상 제작)
- **Gemini Veo 활용 생성**: 스토리보드에 따라 고화질 클립 생성
- **일관성 강화 프롬프팅**: 캐릭터 시트 및 스타일 가이드를 프롬프트에 지속적으로 반영
- **편집 및 합성**: 생성된 클립을 이어 붙이고 전환 효과(Transition) 적용

### Phase 4: Post-Production (후반 작업)
- **오디오 작업**: 전문 내레이션 또는 AI Voice 적용, 배경음악(BGM) 및 효과음 삽입
- **자막 및 그래픽**: 서비스 명칭, 주요 수치 등 텍스트 디자인 추가
- **최종 검수**: 재생 시간(2~3분 준수) 및 전반적인 품질 확인

## 3. 일관성 유지 원칙 (Consistency Principles)

영상이 하나의 완성된 패키지로 느껴지도록 다음 원칙을 엄격히 준수합니다.

| 구분 | 준수 사항 | 관리 방안 |
| :--- | :--- | :--- |
| **비주얼 스타일** | 캐릭터 디자인, 복장, 배경의 아트 스타일 통일 | 공통 스타일 프롬프트(Master Prompt) 사용 |
| **색상 (Color)** | 브랜드 가이드에 따른 메인/서브 컬러 사용 | 특정 HEX 코드값을 프롬프트에 명시 |
| **등장 인물** | 동일한 인물이 영상 전반에 등장 | 캐릭터 일관성(Character Consistency) 기능 활용 및 유사 외모 유지 |
| **메시지 톤** | 전문성 있고 신뢰감을 주는 어조 | 스크립트 작성 시 일관된 문체 사용 |

## 4. 영상 품질 필수 요소 (Quality Elements)

1. **명확성(Clarity)**: 서비스가 해결하는 문제와 해결책이 30초 내에 파악되어야 함
2. **신뢰성(Trust)**: 데이터 기반의 성과나 전문적인 그래픽을 사용하여 서비스의 전문성 강조
3. **가독성(Readability)**: 영상 내 자막은 모바일 환경에서도 잘 보이도록 충분한 크기와 대비 확보
4. **리듬감(Pacing)**: 지루하지 않도록 3~5초 단위로 화면 전환 또는 시각적 변화 부여
